# 코딩테스트 개요
코딩 테스트(Programming Test, Coding Interview)는 지원자의 **문제 해결 능력, 알고리즘 이해도, 코드 구현 능력**을 종합적으로 평가하기 위한 기술 평가 방식이다. 주로 소프트웨어 엔지니어, 데이터 엔지니어, AI 개발자 채용 과정에서 활용된다.  
  
---  
  
## 1. 목적 및 평가 요소

코딩 테스트는 단순히 “코드를 작성할 수 있는가”를 넘어서 다음과 같은 역량을 평가한다:  
  
* **문제 해결 능력 (Problem Solving)**  
  
  * 문제를 분석하고 적절한 접근법을 설계하는 능력  
* **알고리즘 및 자료구조 이해**  
  
  * 시간복잡도, 공간복잡도 고려  
  * 배열, 스택, 큐, 트리, 그래프 등 활용 능력  
* **구현 능력 (Implementation)**  
  
  * 정확하고 효율적인 코드 작성  
* **코드 품질**  
  
  * 가독성, 예외 처리, 논리적 구조  
  
---  
  
## 2. 주요 출제 유형
  
### (1) 자료구조 기반 문제
  
* 배열, 문자열, 해시맵  
* 스택/큐  
* 연결 리스트  
  
### (2) 알고리즘 문제
  
* 정렬 (Quick Sort, Merge Sort)  
* 탐색 (이진 탐색, DFS, BFS)  
* 그리디 알고리즘  
* 동적 계획법(DP)  
  
### (3) 수학 및 구현 문제
  
* 완전 탐색 (Brute Force)  
* 시뮬레이션  
* 문자열 처리  
  
---  
  
## 3. 테스트 형태
  
### (1) 온라인 코딩 테스트
  
* 플랫폼: 프로그래머스, LeetCode, HackerRank  
* 제한 시간 내 문제 해결  
* 자동 채점 시스템  
  
### (2) 오프라인/라이브 코딩 인터뷰
  
* 화상 또는 대면 진행  
* 문제 풀이 + 설명 병행  
* 사고 과정 중요  
  
---  
  
## 4. 평가 기준
  
* **정확성 (Correctness)**: 모든 테스트 케이스 통과 여부  
* **효율성 (Efficiency)**: 시간/공간 복잡도  
* **확장성 (Scalability)**: 큰 입력에서도 동작 가능 여부  
* **코딩 스타일 (Readability)**  
  
---  
  
## 5. 난이도 수준
  
* **기초**: 구현, 문자열 처리  
* **중급**: 자료구조 + 탐색  
* **고급**: 그래프, DP, 최적화 문제  
  
기업별로 난이도 편차가 존재하며, 대기업이나 IT 기업일수록 고난도 문제 비중이 높다.  
  
---  
  
## 6. 준비 전략
  
* **자료구조/알고리즘 이론 학습**  
* **문제 풀이 반복 (패턴 인식)**  
* **시간 복잡도 분석 습관화**  
* **실전 환경 적응 (타이머 설정)**  
  
대표적인 연습 플랫폼:  
  
* 백준 온라인 저지  
* Codeforces  
* 프로그래머스
  
---  
  
## 7. 한계 및 주의점
    
* 실무 능력을 완전히 대변하지는 않음  
* 특정 알고리즘 패턴 암기에 치우칠 위험  
* 긴장 상황에서 실력 발휘 어려움  
  
---  
  
## 요약
  
코딩 테스트는 **이론 + 실전 문제 해결 능력 + 구현 능력**을 동시에 평가하는 기술 검증 도구이며, 특히 컴퓨터공학 기반 직군에서는 필수적인 선발 과정이다.
